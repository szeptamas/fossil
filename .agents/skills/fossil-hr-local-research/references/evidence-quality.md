# Evidence quality

## Highest

- real-watch reproduction;
- fresh build artifact with exact identifier;
- current source controlling the observed behavior.

## Strong

- SDK documentation plus matching working example;
- multiple independent source references;
- current Gadgetbridge implementation plus successful watch test.

## Medium

- one archived issue or PR with technical detail;
- old source that may still apply;
- a community fork with similar behavior.

## Low

- uncited summary;
- old forum claim;
- memory;
- analogy to another watch platform.

Always state uncertainty and design a discriminating test.
